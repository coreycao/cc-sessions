# CC Sessions v1.0.6

This release improves AI-powered session workflows and keeps generated metadata local to CC Sessions.

## Highlights

- Added AI Review caching so unchanged sessions can reuse the previous review result.
- Added AI Review cache metrics to Settings > Data.
- Added local session rename support for Codex threads and Claude Code sessions.
- Added AI-powered title generation using the configured OpenAI-compatible provider.
- Kept renamed titles in CC Sessions metadata instead of modifying Codex or Claude Code source files.
- Improved OpenAI-compatible response parsing for APIs that return array content or top-level choice text.
- Reduced AI Rename input size by sending a compact title context instead of the full transcript.
- Added hard transcript limits for AI Review to reduce long-message token waste.

## Notes

- AI-generated and manually edited session titles are stored locally in CC Sessions app data.
- The original Codex and Claude Code session files are not renamed or edited.
- AI Rename now sends session metadata, early and recent user requests, and short assistant signals rather than the full conversation.
- AI Review still uses conversation content, but long messages and the overall transcript are truncated before sending.
