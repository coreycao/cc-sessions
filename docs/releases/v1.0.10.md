# CC Sessions v1.0.10

This release adds safer AI-assisted organization workflows and continues unifying the app's visual system around quieter loading, button, and error states.

## Highlights

- **Batch AI rename**: select sessions in the session list and generate AI title suggestions in a review dialog. Suggestions are applied only after explicit selection.
- **AI provider guidance**: AI features now detect missing or incomplete provider settings and jump directly to *Settings → AI*, creating an editable provider profile when needed.
- **Shared UI primitives**: common buttons, icon buttons, loading states, overlays, and empty states now use a shared renderer component layer.
- **Cleaner loading experience**: startup now shows a single React loading state instead of flashing both the static boot loader and app loader.
- **Consistent app states**: initial scanning, refresh overlays, empty selections, conversation loading, and app error handling now follow the same visual language.
- **Icon cleanup**: settings navigation uses a more semantically appropriate app-info icon.

## Notes

- Batch AI rename stores titles only in CC Sessions metadata. It never modifies Codex or Claude Code source session files.
- Batch rename defaults to skipping sessions that already have custom titles and generates suggestions for up to 20 pending sessions at a time.
